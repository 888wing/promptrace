# Sync Strategies

Four strategies for synchronizing trace data into project documentation files.

---

## 1. append-managed

**Behavior**: Only modifies content between managed markers. Human content outside markers is NEVER touched.

**Markers**:
```html
<!-- codetape:start:{section-id} -->
{managed content here}
<!-- codetape:end:{section-id} -->
```

**Valid section IDs**: `architecture`, `components`, `recent-changes`, `config`

**Rules**:
- If markers exist: replace content between them, leave everything else untouched
- If markers do not exist: append markers + content at the end of the file
- If only the start marker exists (missing end): warn and skip -- do not write
- If markers are incorrectly nested: warn and skip -- do not write
- Never reorder, delete, or modify any content outside the markers

**Used for**: README.md, CLAUDE.md, AGENTS.md

**Example**:
```markdown
# My Project

Some hand-written intro that will never be touched.

<!-- codetape:start:components -->
| Component | Type | File |
|-----------|------|------|
| AuthService | service | src/services/auth.ts |
<!-- codetape:end:components -->

More hand-written content below, also untouched.
```

---

## 2. prepend

**Behavior**: Adds new content at the top of the file, after any title or front-matter header. Does not remove or modify existing entries.

**Rules**:
- Detect the first `# Title` line or YAML front-matter block (`---` ... `---`) and insert after it
- If no title or front-matter exists, insert at line 1
- Never remove existing entries -- the file grows over time
- New entries are separated from existing content by a blank line

**Used for**: CHANGELOG.md

**Example**:
```markdown
# Changelog

## [0.3.0] - 2026-03-20     <-- newly prepended
### Added
- Payment pre-authorization flow

## [0.2.0] - 2026-03-15     <-- previously existing
### Added
- User authentication
```

---

## 3. overwrite

**Behavior**: Regenerates the entire file from scratch. Previous content is completely replaced.

**Rules**:
- Read current file to understand existing structure (for context only)
- Generate new content from component-map.json and recent traces
- Write the complete file -- no merging, no preservation
- Always include a generation timestamp comment at the top:
  ```
  <!-- Generated by Codetape on YYYY-MM-DD HH:MM. Do not edit manually. -->
  ```

**Used for**: docs/ARCHITECTURE.md, docs/components/*.md

---

## 4. merge

**Behavior**: Reads existing content, identifies sections that need updating, and preserves human-authored content while updating factual sections.

**Rules**:
- Identify sections by heading structure (`##`, `###`)
- For each section, determine if it contains factual/generated content or human-authored narrative
- Update factual sections (component lists, architecture descriptions, file references)
- Preserve human-authored sections (design rationale, usage guides, examples)
- If uncertain whether a section is human-authored or generated, fall back to `append-managed`
- Add a comment to any section that was auto-updated:
  ```
  <!-- Last updated by Codetape: YYYY-MM-DD -->
  ```

**Used for**: Mixed documents with both human and generated content

---

## General rules (apply to all strategies)

1. **Diff preview**: Always show a diff preview of proposed changes before writing
2. **User confirmation**: Never write without user confirmation, unless the `--yes` flag is explicitly passed
3. **Malformed markers**: If `append-managed` markers are malformed (missing end tag, wrong nesting, duplicate IDs), warn the user and skip that section entirely -- do not attempt to fix markers automatically
4. **Sync target configuration**: Sync targets and their assigned strategies are defined in `.codetape/config.json` under the `sync_targets` key:
   ```json
   {
     "sync_targets": [
       { "file": "README.md", "strategy": "append-managed", "sections": ["components", "recent-changes"] },
       { "file": "CHANGELOG.md", "strategy": "prepend" },
       { "file": "docs/ARCHITECTURE.md", "strategy": "overwrite" }
     ]
   }
   ```
5. **Missing files**: If a sync target file does not exist, create it with appropriate content. Do not silently skip.
6. **Encoding**: All files are read and written as UTF-8. Do not alter line endings unless the project has a `.editorconfig` specifying otherwise.
